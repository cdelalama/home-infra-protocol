- 2026-05-01 - Claude - Patch 0.1.5: three GPT-5-review fixes against the freshly published 0.1.4. (1) `integrations/dockit/new-homelab-project.sh` was splitting `--description` with spaces into multiple positional args to `gh repo create` because `DESC_ARG="--description $DESCRIPTION"` was constructed and expanded unquoted. Rewritten to use POSIX positional parameters: `set -- "$GH_OWNER/$PROJECT_NAME" "--$VISIBILITY" "--source=$TARGET_DIR" --remote=origin --push` then conditionally `set -- "$@" --description "$DESCRIPTION"` if non-empty, then `gh repo create "$@"`. (2) `integrations/dockit/checklists/PROJECT_CHECKLIST.md` previously listed `docker compose restart edge-caddy` or `caddy reload` as equivalent options for the edge-caddy reload step. Both wrong for QNAP NAS — `docker compose` subcommand is not on PATH there per `home-infra/docs/CONVENTIONS.md`, and `caddy reload` may not pick up newly-added vhosts depending on how Caddy was started. Replaced with the full compose-plugin path (`/usr/local/lib/docker/cli-plugins/docker-compose -f /share/Container/compose/edge-caddy/docker-compose.yml restart edge-caddy`) and a note preferring `restart` over `reload` for new vhosts. (3) `integrations/dockit/templates/AGENTS.md` mandated PROJECTS.md updates "whenever a project is created, bumps version, or changes status" — too aggressive for projects with frequent internal patches. Tightened to **deployed-reality** scope: project created or retired, deployed version on a host changes, status / host / exposure / URL changes. Internal patches that never reach a host do NOT warrant a PROJECTS.md update — the table tracks observed deployed state, not repo HEAD. End-to-end smoke test against `/tmp/smoke-homelab` confirmed: branch is `main`, AGENTS.md reflects the new wording, checklist reflects the QNAP path.
